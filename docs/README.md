# Docs

This directory organizes product and delivery documentation by stage.

## Structure

### `00-context`
- Project background
- Business context
- Problem statement

### `01-requirements`
- Goals
- Scope
- Functional and non-functional requirements

### `02-user-stories`
- User roles
- User stories
- Acceptance criteria

### `03-spec`
- Detailed product or system specifications
- API, data, and workflow definitions

### `04-architecture`
- Technical architecture
- System boundaries
- Integration and deployment views

### `05-design`
- UX/UI design
- Interaction flows
- Visual design references

### `06-tasks`
- Delivery plan
- Milestones
- Execution checklist
